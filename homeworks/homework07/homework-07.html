Server Security as a Random Walk

Analysis: Two Security Models

This simulation compares two models for a server's security over n weeks.

Model 1: Constant Attackers (Classic Random Walk)

  1. Step Probability (Constant)
    • The number of attackers m is constant.
    • The probability of remaining secure in any given week is q = (1 - p)m.
    • This probability q for a "Secure" (+1) step is the same for all n weeks.
    • The probability of a "Breach" (-1) step is 1 - q.
  2. Convergence (Binomial Distribution):
    • Let k be the number of secure weeks. The final score is S = 2k - n.
    • Since all n trials are independent and identical (same q), the number of successes k follows a Binomial Distribution: k ~ Bin(n, q).
    • As n diverges, this shape converges to a Normal (Gaussian) Distribution (Central Limit Theorem).


Model 2: Decreasing Attackers (Non-Stationary Random Walk)

  1. Step Probability (Variable):
    • The number of attackers in week w is mw = max(0, m_initial - w + 1), since it receives weekly securiy updates.
    • The probability of remaining secure changes every week: qw = (1 - p)mw.
    • qw (the probability of a +1 step) increases as the number of attackers decreases.
  2. Convergence (Poisson Binomial Distribution):
    • The final score is still S = 2k - n, where k is the total number of secure weeks.
    • However, k is now the sum of n trials that are not identical (qw is different each week).
    • This no longer follows a simple Binomial Distribution. It follows a Poisson Binomial Distribution.
    • As n diverges, this distribution also converges to a Normal (Gaussian) Distribution, but with a different mean and variance than Model 1.
